// WHAT IT DOES

스킬 하나, 세 가지 일

무거운 세부는 references/로 분리해 에이전트가 필요한 것만 읽습니다. Anthropic Agent Skills의 progressive disclosure.

01

전체 CLI

auth · daemon · workspace · issue · project · agent · skill · squad · autopilot · runtime. 영역별 전 명령어를 공식 문서와 대조한 정확한 플래그로 담았습니다.

02

온보딩

설치 → setup → 런타임 확인 → 에이전트 생성 → 첫 작업 할당. 빈 머신에서 에이전트가 일을 집어들기까지 다섯 단계.

03

워크플로우

고정 상태 위에서 단계마다 새 컨텍스트로 도는 다단계 파이프라인. 실패하면 work로 rewind. advance/rewind 셸 레시피 포함.

// HOW IT WORKS

작동을 지배하는 세 가지 규칙

보드의 멘탈 모델은 작습니다. 나머지 동작은 모두 이 세 규칙에서 나옵니다.

// 멘탈 모델
workspace
└ project
  └ issue · MUL-123
    ├ status    backlog → done (고정 7단계)
    ├ priority  urgent … none
    ├ metadata  key=value (자유 채널)
    └ assignee  사람 또는 에이전트
01

할당이 곧 디스패치

이슈를 에이전트에 할당하면 데몬이 claim해 도구를 자동 실행합니다. backlog 상태면 발화 없이 대기.

02

@멘션으로 재실행

코멘트에 @에이전트를 적으면 그 이슈에서 새 실행이 트리거됩니다.

03

상태는 고정, metadata는 자유 채널

7개 상태는 추가할 수 없고 라벨 CLI도 없습니다. 단계와 지속 상태는 metadata set key=value에 기록하고 list --metadata로 조회합니다.

// INSTALL

설치

Multica에 바로 import하거나, 번들 설치 스크립트로 어떤 하네스에도 넣을 수 있습니다.

# Multica는 GitHub에서 스킬을 바로 import합니다
multica skill import --url https://github.com/cskwork/multica-skill
multica skill list | grep multica
multica agent skills <agent-slug>   # 에이전트에 부착
git clone https://github.com/cskwork/multica-skill ~/.multica-skill
cd ~/.multica-skill
./install.sh                 # claude / codex / gemini / opencode / pi 자동 감지
./install.sh claude-code     # 또는 하나만 → ~/.claude/skills/multica/
// CLI

CLI 치트시트

표면적의 일부입니다. 스킬은 전체 레퍼런스를 담고 있습니다.

인증 · 데몬

multica setup multica auth status multica daemon start --foreground multica daemon logs

이슈

multica issue create --title … multica issue assign <id> --to … multica issue status <id> in_progress multica issue comment add <id> --content …

에이전트 · 스킬

multica agent create multica agent list multica skill import --url … multica agent skills <slug>

메타데이터 · 실행

multica issue metadata set <id> --key … multica issue list --metadata k=v multica issue runs <id> multica issue pull-requests <id>

프로젝트 · 스쿼드

multica project create --title … multica squad create --leader …

오토파일럿 · 런타임

multica autopilot create --mode … multica autopilot trigger-add <id> --cron … multica runtime list
// PIPELINE

파이프라인

상태는 고정이고 라벨 CLI가 없어, 각 단계를 이슈 metadata(pipeline_status)에 기록합니다. 상태를 옮기고, 다음 에이전트를 재할당하고, 진행합니다. 단계마다 새 컨텍스트.

explore→ work→ review→ qa→ learn→ done
↩review · qa에서 문제 발견 → work로 rewind. 에이전트는 그 지적만 읽고 고칩니다.
✕QA 3회 연속 실패 → blocked + 구독자(사람) 호출.
# 한 단계 진행하고 다음 에이전트를 디스패치
multica issue metadata set MUL-142 --key pipeline_status --value review
multica issue status MUL-142 in_review
multica issue assign MUL-142 --to claude-reviewer
