Install

Import into Multica directly, or drop it into any harness with the bundled installer.

Via Multica
Any harness
# Multica has native skill import from GitHub
multica skill import --url https://github.com/cskwork/multica-skill
multica skill list | grep multica
multica agent skills <agent-slug>   # attach to an agent
git clone https://github.com/cskwork/multica-skill ~/.multica-skill
cd ~/.multica-skill
./install.sh                 # auto-detects claude / codex / gemini / opencode / pi
./install.sh claude-code     # or target one → ~/.claude/skills/multica/

The pipeline

States are fixed and there's no label CLI — so each phase lives in issue metadata (pipeline_status). Move status, reassign, advance. Fresh context every phase.

explore→ work→ review→ qa→ learn→ done

review or qa finds a problem → rewind to work · 3 QA fails → blocked + ping a human

# advance one phase and dispatch the next agent
multica issue metadata set MUL-142 --key pipeline_status --value review
multica issue status MUL-142 in_review
multica issue assign MUL-142 --to claude-reviewer
