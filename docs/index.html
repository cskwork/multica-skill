전체 Multica CLI를
담은 하나의 Agent Skill

어떤 코딩 에이전트에게든 전체 multica 명령어, 머신 온보딩, 그리고 보드 위 explore→work→review→qa→learn 파이프라인을 가르칩니다.

스킬 설치하기 GitHub에서 보기 →
multica skill import --url https://github.com/cskwork/multica-skill

스킬 하나, 세 가지 역할

무거운 세부 내용은 references/로 분리해 에이전트가 필요한 것만 읽습니다 — Anthropic Agent Skills의 progressive disclosure.

⌘

전체 CLI

auth · daemon · workspace · issue · project · agent · skill · squad · autopilot · runtime — 영역별 전 명령어를 공식 문서와 대조한 정확한 플래그로.

⊕

온보딩

설치 → setup → 런타임 확인 → 에이전트 생성 → 첫 작업 할당. 빈 머신에서 에이전트가 일을 집어들기까지 5단계.

⟳

워크플로우

Multica의 고정 상태 위에서 단계마다 새 컨텍스트로 도는 다단계 파이프라인. 실패하면 rewind. advance/rewind 셸 레시피 포함.

설치

Multica에 바로 import하거나, 번들 설치 스크립트로 어떤 하네스에도 넣을 수 있습니다.

Multica 경유
다른 하네스
# Multica는 GitHub에서 스킬을 바로 import합니다
multica skill import --url https://github.com/cskwork/multica-skill
multica skill list | grep multica
multica agent skills <agent-slug>   # 에이전트에 부착
git clone https://github.com/cskwork/multica-skill ~/.multica-skill
cd ~/.multica-skill
./install.sh                 # claude / codex / gemini / opencode / pi 자동 감지
./install.sh claude-code     # 또는 하나만 → ~/.claude/skills/multica/

CLI 치트시트

표면적의 일부입니다. 스킬은 전체 레퍼런스를 담고 있습니다.

인증 & 데몬

multica setup multica auth status multica daemon start --foreground multica daemon logs

이슈

multica issue create --title … multica issue assign <id> --to … multica issue status <id> in_progress multica issue comment add <id> --content …

에이전트 & 스킬

multica agent create multica agent list multica skill import --url … multica agent skills <slug>

메타데이터 & 실행

multica issue metadata set <id> --key … --value … multica issue runs <id> multica issue pull-requests <id>

프로젝트 & 스쿼드

multica project create --title … multica squad create --name … --leader …

오토파일럿

multica autopilot create --mode create_issue multica autopilot trigger-add <id> --cron …

파이프라인

상태는 고정이고 라벨 CLI가 없어, 각 단계를 이슈 metadata(pipeline_status)에 기록합니다. 상태를 옮기고, 재할당하고, 진행합니다. 단계마다 새 컨텍스트.

explore→ work→ review→ qa→ learn→ done

review나 qa에서 문제 발견 → work로 rewind · QA 3회 실패 → blocked + 사람 호출

# 한 단계 진행하고 다음 에이전트를 디스패치
multica issue metadata set MUL-142 --key pipeline_status --value review
multica issue status MUL-142 in_review
multica issue assign MUL-142 --to claude-reviewer
